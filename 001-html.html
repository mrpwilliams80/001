001-html

all tags for html5 presented in alphabetical order by usage catagory with examples. Many tags in the html5 specification are used to provide somantic meaning to the elements of the page. The reason for this is three fold:

  • to reduce and over use of the <div> element
  • help the browser understand the meaning of content rather than just display it
  • help search engines to read and find the required information faster

    • While it is still posssible (and sometimes neccesaary) to use <div> tags, whever an alternative tag would provide more meaning it is recomended to use that instead.

    Definitions come from TutorialRepublic [Feb 2020].

    Structural Tags

    <a>
    - The a in this tag is short for 'anchor'. Defines a hyperlink for web navigation.
    example <a href="http://www.google.com">a link to google</a> gives →

    <article>
    - This tag is used to define articles. It does not have any specific formating. Rather, it is used to give somantic meaning to html code

    <body>

    Metadata Tags

    <base>
    - Defines the base URL for all linked objects on a page. Provides a common target to for all relative URLs contained within a document. There must be no more than one <base> tag per document.
    example <base href="https://www.tutorialrepublic.com/"> <a href="css-tutorial/">Learn CSS</a> gives →
    Learn CSS
    <link>
    - Defines the relationship between the current document and an external resource. This element may only appear in the head section of a document, although it may appear any number of times. It is mostly used in linking style sheets to the HTML document.
    example <link rel="stylesheet" href="default.css"> would link to a css style sheet named 'default.css' in the same directory as the html file that contains it.
    <meta>
    - Provides structured metadata about the document content. This element may only appear in the head section of a document, although it may appear any number of times. It is commonly used to provide keywords, descriptions, author information, and other metadata that may be used by the browser to render the document correctly, or it may be used by the search engines for indexing purposes.
    example <meta charset="UTF-8"> → Define the character encoding for the document
    <meta name="author" content="John Smith"> → Defines the author name of the document
    <meta name="keywords" content="HTML,CSS,Tutorials,References,Examples"> → Defines a list of keywords for search engines
    <meta name="description" content="HTML/CSS Tutorials and References"> → Define a brief summery/description of the web page
    <meta name="viewport" content="width=device-width, initial-scale=1"> → Enable zooming in mobile devices
    <style>
    - Inserts style information (commonly CSS) into the head of a document. This tag is used to define style rules at a page-level (i.e. within the document). The rules inside the element tell a browser how to present the document. You can place any number of style elements in a document, but it must appear inside the head section
    example <style type="text/css">h1 {color: #993434;}</style> would set an h1 elements colour as shown →

    Colour changed to #993434

    <title>
    - Defines a title for the document. This element should be the first element defined in the head section of the document. The title should be meaningful because it summarizes the content or purpose of the document.
    example <title>Title</title> would set the title of a page to 'Title'

    Table Tags

    Tables should only be used for tabular information. If you just want to organise elements into a grid arrangement css grid or flexbox are more appropriate.

    <caption>
    - Defines the title of a table. The caption text should describe the nature of the table. The caption tag only permitted immediately after the <table> start tag. A <table> element may only contain one <caption> element.
    <table>
    - Defines a data table. It is used to represents data in a grid-like fashion (in rows and columns). However, if the data is not tabulated you should consider grid or flexbox in css.
    <tbody>
    - Groups a set of rows defining the main body of the table data. Short for table body, this element defines a table's body. It is used to group a set of rows defining the main body of the table data. It must appears as a child of a <table> element, after any <caption> and <thead> elements, and must contain at least one row, defined by the <tr> element.
    <td>
    - Defines a cell in a table. Short for table data, this tag defines a data cell in a table. A Table cells may contain two types of information: header information and data.
  • The <th> element defines a cell that contains header information.
  • The <td> element defines a cell that contains data.
    • This distinction enables web browser to render header and data cells distinctly, even in the absence of style sheets. For example, header cell text rendered with a bold font while text inside the data cell rendered as a regular text.
    <tfoot>
    - Groups a set of rows summarizing the columns of the table. Short for table foot, this defines a table's footer. It is used to group a set of rows summarizing the columns of the table. It must appears as a child of a <table> tag, after any <caption> and <thead> elements, but before any <tbody> and <tr> element. The <tfoot> element must contain at least one row, defined by the tag.
    <thead>
    - Groups a set of rows that describes the column labels of a table. Short for table head, this element defines a table's header. It is used to group a set of rows defining the header of the columns within the table. This element must appears as a child of a <table> element, after any <caption> elements, but before any <tbody>, <tfoot> and <tr> elements. The <thead> tag must contain at least one row, defined by the <tr> tag.
    <th>
    - Defines a header cell in a table. Short for table head, this tag defines a data cell in a table. A Table cells may contain two types of information: header information and data.
  • The <th> element defines a cell that contains header information.
  • The <td> element defines a cell that contains data.
    • This distinction enables web browser to render header and data cells distinctly, even in the absence of style sheets. For example, header cell text rendered with a bold font while text inside the data cell rendered as a regular text.
    <tr>
    - Defines a row of cells in a table. Short for table row, this element defines a row of cells in a table. The element acts as a container for table cells and contains one or more <th> or <td> elements.
    example <table>
        <caption>Table Title</caption>
        <thead>
           <tr>
              <th></th>
              <th>Column 1</th>
              <th>Column 2</th>
              <th>Column 2</th>
           </tr>
        </thead>
        <tfoot>
           <tr>
              <td colspan="4">Copyright &copy; 2020 UnecessaryCopyrightCo.</td>

        <tbody>

              <th>Col1, Row1</th>
              <th>Col2, Row1</th>
              <th>Col2, Row1</th>
           </tr>
           <tr>
              <th>Row 2</th>
              <th rowspan="2">Col1, Rows 2 and 3</th>

              <th>Col2, Row2</th>
           </tr>
           <tr>
              <th>Row 3</th>
              <th>Col2, Row3</th>
              <th>Col2, Row3</th>
           </tr>
        </tbody>
    </table>
